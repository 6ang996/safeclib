27 # General quirks
28 
29 Generally most libc's use a 4 byte wchar_t (which resembles UTF-32),
30 but windows/cygwin/solaris/aix use 2-byte (UTF-16), therefore they need
31 to support surrogate pairs.
32 
33 The wide sprintf variants do not allow a NULL buffer argument to
34 return the size of the resulting buffer. If the initial buffer is too
35 small, you need to realloc and redo.
36 
37 There's still no locale-independent utf8 support, not even in C11 with
38 its new `u8""` type.
39 
40 Many wchar and mb conversions and searches are locale-dependent, hence
41 unusable for utf8.
42 
43 Nobody implements a proper wchar case-conversion, foldcase and
44 normalization API. ICU or libunistring are way too heavy for this
45 simple problem. There's only utf8proc for utf8 encoded strings (now
46 with julia).
47 
48 See also http://crashcourse.housegordon.org/coreutils-multibyte-support.html
49 
50 # C11 Annex K/safec caveats
51 
52 * snprintf, vsnprintf, snwprintf, vsnwprintf, tmpnam_s:
53 
54  They are all considered unsafe. The 4 'n' truncating printf versions
55  don't guarantee null-delimited destination buffers.
56 
57 * `tmpnam_s` and `tmpnam` are racy.
58 
59 ## Microsoft Windows/MINGW_HAS_SECURE_API
60 
61 * `fopen_s`, `freopen_s` deviate in the API: restrict is missing.
62 
63 * `strtok_s`, `wcstok_s`,`vsnprintf_s` miss the dmax argument.
64 
65 * `vsnprintf_s` adds a maxarg argument.
66 
67 * `vswprintf` adds a maxarg argument on w32. (with `__STRICT_ANSI__`
68  undefined)
69 
70 * no `strnlen` on mingw32.
71 
72 * no `errno_t` return type for `qsort_s`, only `void`.
73 
74 * reversed argument order for `localtime_s` and `gmtime_s`.
75 
76 * older mingw versions have `wchar.h` with only 2 functions:
77  `wcscmp`, `wcslen`
78 
79 ## safeclib
80 
81 * safeclib does not check optional NULL parameters to the `scanf_s`,
82  `printf_s` functions. This would need tighter integration into the
83  upstream libc.
84 
85 * safeclib `fgets_s` permits temporary writes of `dmax+1` characters
86  into dest.
87 
88 * does not work fine with 2-byte wchar_t
89  (i.e. cygwin/windows/solaris/aix), esp. the new case converters and
90  `wcsnorm_s`. surrogate pairs are not yet supported everywhere.

92 # Other caveats
93 
94 ## glibc
95 
96 * SEGV with `freopen(NULL, "rb", stdin)` with asan on some systems,
97  calling an invalid `strlen()` on NULL.
98 
99 * quirky declaration of various standards, which conflict with each other.
100 
101  glibc needs the correct standard to include some extensions
102  when we declare the standard by ourselves.
103  e.g. clang-4.0 -std=c99 misses several reentrant versions.
104  when defining `_XOPEN_SOURCE 700` to define `strnlen` and the reentrant
105  time versions, we need also `__STDC_WANT_LIB_EXT1__ 1` for `errno_t`,
106  and this would break several other older struct members, such as `tm_gmtoff`
107 
108 ## newlib
109 
110 * `vswscanf` is broken with a format string containing L"%%n"
111 
112 * The following multibyte API's are missing, and can be defined like
113  this:
114 
115 ```
116  mbstate_t st;
117  #define wctomb(dest, wc) wcrtomb((dest), (wc), &st)
118  #define wcstombs(dest, src, len) wcsrtombs((dest), &(src), (len), &st)
119  #define mbstowcs(dest, src, len) mbsrtowcs((dest), &(src), (len), &st)
120 ```
121 
122 ## FreeBSD libc
123 
124 * `vswscanf` is broken with a format string containing L"%%n"
125 
126 * `mbstowcs` is broken with (NULL, '\0')
127 
128 ## musl
129 
130 * `wmemcmp` returns not -1,0,1 but the full ptr diff.
131 
132 * `mbtowc` and `wctomb` accept and convert illegal 4 byte characters
133  in the ASCII locale to surrogate pairs, as it would be unicode.
134  e.g. it converts `\xa0` to `\xdfa0`.
